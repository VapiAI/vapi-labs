# Contributing

Currently, this repo is open to **Vapi employees only** 

## Before you open a PR

1. Ensure there is no exposure of any keys or data
3. Run it yourself from a clone before opening the PR to ensure it works for visitors.

## Structure your project like this

Copy [`projects/_template/`](projects/_template) as a starting point:

```
projects/<your-project-name>/
├── README.md
├── (optional) AGENTS.md or SKILL.md
├── .env.example
└── (your code)
```

Your `README.md` should include, at minimum:

- One paragraph on what it is and why you built it
- Setup/run instructions that work from a clean clone
- The disclaimer block 
- Known limitations 

## Opening the PR

- One project per PR.
- Fill out the PR template checklist
- all pr's require one reviewer
